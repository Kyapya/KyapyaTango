# 第2段階: Notion自動同期

Notion辞書データベースを原本として、辞書JSON・単語HTML・トップページ・検索索引をGitHub Actionsで自動更新します。

## 初回設定

1. Notionで内部インテグレーションを作成し、辞書データベースへ接続します。
2. GitHubリポジトリの `Settings` → `Secrets and variables` → `Actions` で `NOTION_TOKEN` を登録します。
3. GitHubの `Actions` → `Sync dictionary from Notion` → `Run workflow` を実行します。

データベースに複数のデータソースがある場合だけ、追加で `NOTION_DATA_SOURCE_ID` を登録します。

## 動作

- 6時間ごと、および手動実行時にNotionを確認
- タイトルプロパティ `ALL`、タグ `英単語` のページだけを同期
- 全件同期では `config/notion.json` の `minimum_last_edited_time` 以降に更新されたページだけを対象にする
- 同じ見出し語が複数ある場合は最終更新版を採用
- 全件同期中に解析・検証エラーが出たページはスキップし、残りのページを続ける
- エラーになったページは既存JSONを上書きしない
- 変更がある場合だけGitHub Actions botがコミット

現在の自動同期開始日時は `2026-07-11T00:00:00Z` です。これより前に最後に更新された旧形式ページは、自動の全件同期では処理されません。

## 手動同期

Actionsの `word` 入力へ `post` などを指定すると、その単語だけ同期します。この場合は開始日時より古いページでも対象になり、形式エラーがあればワークフローは失敗として表示します。

`word` を空欄にすると安全な全件同期になります。旧形式ページを除外し、個別ページのエラーがあっても処理を継続します。
